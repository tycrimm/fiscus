Variation A

Tight matrix — single-line cells, dense rows

Same components × windows grid as currently shipped, but each cell is one line ($ then small muted %), row padding cut to a hair. Roughly 1/3 the vertical space of the current implementation. The dense format lets the data speak even when most cells say "—".

  • "—" cells naturally fade because they're tiny — your eye lands on the values.
  • Same data, same shape; just denser.
Variation B

Inverted — windows down, components across

Same data transposed. Windows are rows (like the rail's existing 1D/1W/1M/YTD/1Y change list); components are columns with NW set apart on the right as the column-sum. Reads the way the rail already taught your eye — "what happened in 1D" is one row.

Variation C

Per-window scorecards

Each window is a horizontal panel. Window label on the left, four bucket "chips" inline (label + value), then the NW total set off on the right. Less table-y; reads as "for this window, here's what each bucket did and what the total was."

Variation D

In-rail matrix — replaces the existing change rows

Variation E

Strip per component — bucket rows with windowed cells

Each component (and NW total) is a row with a horizontal strip of 5 windowed cells, each with the window label, $ delta, and small %. Vertical separators between cells. Reads as a per-bucket scorecard rather than a strict matrix.
